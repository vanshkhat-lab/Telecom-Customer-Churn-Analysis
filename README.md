# Telecom Customer Churn Analysis (SQL Project)

## Problem Statement
Why are customers leaving and how can churn be reduced?

## Objective
- Identify churn patterns in telecom customers
- Find high-risk customer segments
- Propose data-driven retention strategies

## Dataset
- Telecom customer subscription dataset
- Contains customer tenure, contract type, services, payment method, and billing details
- ~30,000 customer records

## Tools & Skills Used
- SQL
- GROUP BY
- CASE WHEN
- JOIN
- Subqueries

## Analysis Performed
- Active vs churned customers
- Overall churn rate
- Churn by contract type
- Churn by tenure
- Churn by internet service
- Churn by payment method
- Average monthly charges comparison
- Risk analysis using JOIN

## Key Insights
- Month-to-month contract customers have highest churn
- Customers with tenure less than 1 year churn more
- Fiber optic users show higher churn
- High monthly charges increase churn risk
- Lack of support services leads to churn

## Recommendations
- Promote long-term contracts through discounts
- Improve onboarding experience for new customers
- Optimize pricing for high-value customers
- Bundle tech support and security services
